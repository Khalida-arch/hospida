## Key lessons learned
- Vapi sends many event types during a call — always check the 
  actual value in Make's webhook output before writing filters
- Make filters are case sensitive — use Contains instead of Equal to
  where possible to avoid capitalisation mismatches
- Don't fight the filter — remove it while debugging, add it back 
  once the pipeline is confirmed working
- The transcript field at message.artifact.transcript contains the 
  full conversation including the function call parameters Soji read out
- Replay run in Make is a huge time saver — no need to redo test 
  calls when debugging field mappings
